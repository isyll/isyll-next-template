# Architecture Decision Records

Short, dated records of significant, hard-to-reverse decisions and **why** they
were made — context that isn't obvious from the code. Read these before
proposing changes to the foundation.

Format: Status · Context · Decision · Consequences. Number sequentially; never
rewrite history — supersede an ADR with a newer one and link them.
